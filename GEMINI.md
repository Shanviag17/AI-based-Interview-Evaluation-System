# InterviewIQ AI

InterviewIQ AI is a full-stack web application designed to provide personalized mock interview experiences. It uses AI (Hugging Face LLM) to analyze a candidate's resume and generate tailored interview questions, providing a score and feedback based on their responses.

## Project Structure

- **backend/**: FastAPI-based Python server.
  - `main.py`: Entry point with logic for PDF processing, dynamic question generation via Hugging Face, and evaluation.
  - `key.env`: Configuration file for environment variables (e.g., `HF_TOKEN`).
  - `resume.pdf`: Temporary storage for uploaded resumes.
  - `venv/`: Python virtual environment.
- **frontend/**: React-based frontend using Vite.
  - `src/App.jsx`: Main application component with UI logic and styling.
  - `package.json`: Frontend dependencies and scripts.

## Technology Stack

### Backend
- **FastAPI**: Web framework for high-performance APIs.
- **Hugging Face Inference API**: Used for dynamic question generation with the `google/gemma-4-31B-it:novita` model.
- **PyPDF2**: Extracts text from candidate resumes.
- **scikit-learn**: Performs TF-IDF analysis for answer evaluation.
- **python-dotenv**: Manages environment variables.

### Frontend
- **React (v19)**: UI library.
- **Vite (v8)**: Development and build tool.
- **Axios**: HTTP client for backend communication.

## Building and Running

### Prerequisites
- Python 3.x
- Node.js (v18+)
- Hugging Face API Token

### Backend Setup
1. Navigate to `backend/`.
2. Activate virtual environment:
   ```bash
   .\venv\Scripts\activate  # Windows
   ```
3. Install dependencies:
   ```bash
   pip install fastapi uvicorn PyPDF2 scikit-learn pydantic python-dotenv huggingface_hub
   ```
4. Configure `key.env`:
   Add your Hugging Face token:
   ```env
   HF_TOKEN=your_huggingface_token_here
   ```
5. Start the server:
   ```bash
   uvicorn main:app --reload
   ```

### Frontend Setup
1. Navigate to `frontend/`.
2. Install dependencies: `npm install`
3. Start development server: `npm run dev`

## Development Conventions

- **Dynamic Questions**: Generated by `google/gemma-4-31B-it:novita` based on resume content and target role.
- **Evaluation Logic**: Combines TF-IDF similarity (relevance) with answer length (depth) to score responses.
- **Styling**: Modern, responsive UI defined via inline styles in `App.jsx`.
